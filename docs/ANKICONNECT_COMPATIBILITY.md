# AnkiConnect Compatibility

RemNoteConnect includes an opt-in, clean-room AnkiConnect v6 compatibility listener. It implements the public protocol independently in TypeScript and copies no GPL implementation code.

## Enable it

Compatibility Mode is disabled by default. Keep RemNoteConnect read-only until you intend to allow an existing AnkiConnect client to write.

```sh
REMNOTE_CONNECT_ANKI_COMPAT=on \
REMNOTE_CONNECT_ANKI_API_KEY='choose-a-local-secret' \
npx pnpm@11.7.0 --filter @remnoteconnect/daemon start
```

The listeners are separate:

- Native RemNoteConnect: `http://127.0.0.1:8766/`
- AnkiConnect compatibility: `http://127.0.0.1:8765/`

The compatibility listener accepts the official `action`, `version`, `params`, and optional `key` fields. Existing clients that support an AnkiConnect API key should send the value configured in `REMNOTE_CONNECT_ANKI_API_KEY`.

To permit mutations, explicitly turn off native read-only mode through the authenticated native API:

```sh
node scripts/rnc.mjs readonly off
```

Turn it back on after the write window.

## Compatibility status

The pinned contract contains 122 actions:

- 3 native protocol actions
- 31 RemNote-translated actions
- 37 compatibility-sidecar actions
- 51 capability-blocked actions

The 71 implemented actions cover protocol/reflection/batching, profiles, decks and deck configurations, note creation and updates, tags, supported queries, note/card identity lookups, models, templates, fields, styling, and media.

The 51 blocked actions are still recognized and return a deterministic string error. They are blocked because RemNote does not expose faithful equivalents for:

- Anki collection review logs and collection-stat HTML
- Explicit collection sync/reload acknowledgements that RemNoteConnect cannot verify
- Anki scheduler state and writes, including ease, due dates, suspension, intervals, relearning, and answering
- `cardsInfo` and `cardsModTime`, whose official schemas require card state that RemNote cannot supply honestly
- Anki desktop GUI selection, reviewer, browser, import, undo, and shutdown controls
- APKG import/export
- Destructive deck/tag cleanup whose RemNote semantics differ materially

Call the native `capabilities` action for the full machine-readable manifest and each action’s current status. Call AnkiConnect `apiReflect` for the official action-name surface.

## Important semantic mappings

- Anki note → practice-enabled Rem
- Anki card → native card generated by RemNote from that Rem
- Anki deck hierarchy → RemNote folder/document path
- Anki tags → RemNote tags
- Anki models/templates/configs → local compatibility sidecar metadata
- Anki integer IDs → persistent public IDs mapped to RemNote string IDs

IDs are stable across daemon restarts and are never recycled after deletion.

Model/template and deck-configuration actions round-trip compatibility metadata for client interoperability. They do not replace RemNote's native card renderer or scheduler, so changing an Anki template or deck interval does not change how RemNote schedules a card.

## Query support

Deck, tag, note-ID, and plain-text searches are translated. Query clauses that would be broadened or misinterpreted—such as scheduler properties, rated/added windows, negation, grouping, and `OR`—return an error instead of an inaccurate result.
